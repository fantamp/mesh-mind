# Структура Проекта Mesh Mind

Этот файл определяет высокоуровневую структуру каталогов проекта. Агенты должны придерживаться этой архитектуры.

```text
.
├── AGENTS.md                   # Правила для AI-агентов
├── ...                         # Корневые конфиги (README, task.md, requirements.txt, .env)
│
├── specs/                      # Атомарные спецификации (Specs)
│
├── ai_core/                    # Основной сервис приложения (The Core)
│   ├── api/                    # Точки входа API (FastAPI)
│   ├── ui/                     # Админ-панель и визуализация (Streamlit)
│   ├── agents/                 # Логика AI-агентов (Google ADK)
│   ├── rag/                    # Работа с базой знаний и векторами
│   └── common/                 # Общие утилиты, конфиги и модели данных
│
├── cli/                        # CLI-инструменты для ручных операций (импорт данных)
│
├── n8n_scripts/                # Скрипты для интеграции в n8n (Router, Parsers)
│
├── tests/                      # Тесты (Unit & Integration)
│
└── data/                       # Локальное хранилище данных (в .gitignore)
    ├── db/                     # Реляционная БД (SQLite)
    ├── vector_store/           # Векторная БД (ChromaDB)
    └── media/                  # Файловое хранилище (Docs, Images, Voice)
```

## Пояснения

1.  **`ai_core/`**: Сердце приложения. Содержит всю бизнес-логику, API и UI. Названо так, чтобы отражать суть сервиса.
2.  **`n8n_scripts/`**: Скрипты здесь служат "источником правды" для кода, который копируется в n8n.
3.  **`cli/`**: Скрипты для запуска человеком (one-off tasks).
4.  **`data/`**: Хранилище состояния. Должно быть исключено из git.

# New Commit Notification

## Overview
Автоматическое уведомление в чат о появлении новых коммитов в удаленном репозитории.

## Problem Statement
Команда и администраторы не знают оперативно о том, что в репозиторий был запушен новый код. Это приводит к тому, что обновления на продакшн-сервере накатываются с задержкой, и теряется синхронизация между разработкой и эксплуатацией.

## Solution Statement
Реализовать фоновую задачу (background task), которая периодически (например, раз в 15 минут) проверяет наличие новых коммитов в удаленном репозитории.
- Если обнаружены новые коммиты, бот отправляет уведомление в админский чат.
- Чтобы избежать спама, система запоминает ID последнего увиденного коммита и уведомляет только о тех, которые появились после него.

## Value Statement
1. **Синхронизация:** Команда сразу узнает о доступных обновлениях.
2. **Скорость:** Ускоряет процесс доставки изменений (CD) на продакшн, так как админ получает триггер "пора обновляться".

## Architecture / Technical details
- **Расположение:** На первом этапе реализуется внутри модуля `telegram_bot` (например, как `asyncio` задача при старте бота). В будущем может быть вынесено в отдельный сервис.
- **Логика:**
    - `git fetch` для получения данных с remote.
    - Сравнение `HEAD` (или локальной ветки) с `origin/main`.
    - Фильтрация уже увиденных коммитов по их Hash ID.
- **Периодичность:** Раз в 15 минут.

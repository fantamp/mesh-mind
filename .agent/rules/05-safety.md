# Безопасность и Git

## Секреты и Ключи

- **НЕ коммитить** API ключи, токены, пароли
- **Использовать** `.env` для конфиденциальных данных
- **Проверять** `.gitignore` перед коммитом

## Legacy Code

- После создания решения проверяй, что не оставил legacy кода
- Удаляй неиспользуемые зависимости из `requirements.txt`

## Git Workflow

- Перед коммитом запускай проверки (см. workflow `pr-check.md`)
- Убедись, что `.env` в `.gitignore`

# Требования к Тестированию

## Общий Принцип

После реализации функциональности **обязательно** создавать тесты.

## Тестирование AI-Агентов

### 1. Обязательные Evaluation Тесты

**При создании агентов через ADK обязательно пиши evaluation тесты** для запуска через `adk eval`.

Структура eval тестов:

```
tests/agents/eval/
├── orchestrator/
│   └── routing_eval.yaml
├── summarizer/
│   └── quality_eval.yaml
└── chat_observer/
    └── search_eval.yaml
```

**Когда создавать**:
- Создал нового агента → создай eval тесты
- Изменил поведение агента → обнови eval тесты

**Документация**: https://google.github.io/adk-docs/evaluate/

### 3. Unit Тесты (pytest)

Для тестирования отдельных компонентов используй `pytest`:

- Отдельные функции и классы
- Интеграция между компонентами
- Граничные случаи

**Команда**: `pytest tests/`

## Стратегия Тестирования

1. **Разработка агента** → ручное тестирование через `adk web`
2. **После отладки** → создание eval тестов (обязательно!)
3. **Для компонентов** → unit тесты через pytest
4. **CI/CD** → автоматический запуск `adk eval` и `pytest`
